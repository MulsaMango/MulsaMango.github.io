# Resume Writing Skill Design

## Purpose

Create a project skill at `.cursor/skills/resume-writing/SKILL.md` that helps draft and refine resume content for this portfolio.

The skill should be most useful when turning rough notes into polished resume sections: profile summaries, experience bullets, skills lists, role descriptions, and targeted wording options.

## Default Positioning

Unless the user says otherwise, the skill should position Tulsa as a Product Designer with a UI and Design Systems focus.

The resume should stay broad enough for product design roles while making the differentiators clear:

- UI craft
- Design systems
- Product design
- Complex B2B and enterprise product contexts
- AI-enabled design workflows
- Design-to-code fluency
- Process improvement and documentation

## Audience

Optimise for a balance between recruiter scannability and design-lead credibility.

The resume should be concise and easy to skim, but not so high level that it loses the substance of the work. It should give design leaders enough specificity to understand scope, judgement, and maturity.

## AI-Enabled Angle

AI-enabled design work should be a focal thread, not a hype layer.

Frame AI as practical, hands-on fluency across design workflows:

- using tools like Figma Make, Cursor, Claude Code, GitHub Copilot, and Codex
- accelerating exploration, prototyping, drafting, and documentation
- bridging design systems, product context, and implementation constraints
- experimenting with emerging workflows while keeping design judgement in charge

Avoid implying that AI replaces design thinking, craft, or collaboration.

## Relationship To Case Study Skill

The resume skill should intentionally reuse the useful writing preferences from `case-study-building`, adapted for resume context:

- Australian English
- no em dashes or en dashes
- no AI-generated filler
- no dramatic rhetorical pivots
- no LinkedIn influencer phrasing
- no “dive deep”, “leverage”, “holistic”, “synergy”, "delve" or similar corporate filler
- confident without being boastful
- specific rather than generic
- concise, but not empty
- human and direct, but not casual or chatty
- professional, but not stiff or overly formal

The case study skill’s “conversational with substance” principle should become “plain, direct, human, and professional” for resume writing.

## Drafting Behaviour

When drafting from rough notes, the skill should usually provide 2-3 concise wording options with different emphasis.

Useful option types:

- recruiter-readable version
- design-systems-focused version
- AI-enabled workflow version
- stronger product-design framing
- more concise version

The skill should ask for missing context when a draft would otherwise become vague, especially for dates, scope, measurable outcomes, role ownership, tools, or target role.

## Resume Content Rules

Experience bullets should usually include:

- action
- scope or context
- outcome or value

Prefer concrete verbs and grounded phrasing. Do not inflate impact. If an outcome is not measurable, frame it honestly through contribution, enablement, clarity, consistency, speed, quality, or adoption signals.

Avoid generic responsibility bullets like “worked on design systems” unless they are expanded with context and value.

## Skill Structure

The skill should include:

- frontmatter with `name: resume-writing`
- a specific trigger-focused description
- purpose and default positioning
- clarifying questions before drafting
- resume voice and style rules
- AI-enabled positioning guidance
- bullet-writing guidance
- output format preferences
- anti-patterns
- a few compact examples based on the current resume draft

No supporting files are needed initially.
